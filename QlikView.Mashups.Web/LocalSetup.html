Mashups with QlikView - Setup for Offline Use

Hints for Setting up the Examples on your Local Machine:

Fortunately there is - in most cases - only little work to get the examples up and running.

  • Prerequisites for running this microsite offline:
    • A local installation of QlikView Accesspoint + QlikView Server
    • The local QlikView Accesspoint has to be accessible (e.g. via http://localhost)
    • Ensure that the following link works: http://localhost/QvAJAXZfc/opendoc.htm?document=Sales%20Compass.qvw
    • If not, download the "Sales Compass.qvw" application from demo.qlikview.com and make it accessible in the root directory of your Accesspoint
  • Demo Applications
    • There are two applications used within this microsite, the "What's new in QlikView11" and "Sales Compass"
    • By default both applications should be installed on your computer if you have installed QlikView Desktop
    • Make sure that these applications are available on your machine (or download them from demo.qlikview.com)
  • Download the application
      • The latest release of this solution can be found at https://qvmashups.codeplex.com/releases
  • Set Up Web Server
    • Create a virtual directory on your local web server (e.g. find some instructions for Internet Information server here) and copy the downloaded content into that directory.
    • Note: This microsite will also work without a web server, but in this case I'd recommend to use Internet Explorer or Firefox instead of Chrome
  • Configuration
    • Go to the directory "library/js/".
    • There you'll find a file called "Config.js"
    • Open this file in any text editor
    • Comment the four lines of the "ONLINE CONFIGURATION"
    • Uncomment the four lines for the "LOCAL CONFIGURATION"
    • The configuration should then look like this example (or similar)
    • Save the file
